# 世足主題網頁足球遊戲 — 產品需求文件 (PRD)

> 版本：v1.0 ｜ 狀態：規劃中 ｜ 最後更新：2026-06-10

---

## 1. 產品概述

### 1.1 一句話定位
搭上世足賽熱潮的純網頁（無需安裝）足球小遊戲合輯，包含三種玩法，主打「即點即玩、分享拚分數」。

### 1.2 目標
- 蹭世足賽流量，做成可在手機 / 桌機瀏覽器直接玩的輕量遊戲。
- 三個模式各自獨立，難度與互動方式不同，提升重玩率。
- 強調「物理手感」與「分享排行」兩個記憶點。

### 1.3 平台與相容性
| 項目 | 需求 |
|------|------|
| 平台 | 純前端網頁（行動裝置優先 Mobile-first） |
| 輸入 | 桌機滑鼠 / 觸控 / 模式 2 需攝影機 |
| 瀏覽器 | Chrome、Safari（iOS）、Edge 最新兩個版本 |
| 方向 | 直式為主，支援橫式 |
| 網路 | 純前端可離線玩；**無線上排行榜**（分數僅存本機 localStorage） |
| 語系 | **多語系（中／英）** |

## 2. 共通需求

### 2.1 視覺主軸
- 視覺風格：**扁平卡通 2D（Flat / Cartoon）起步**，工時低、手機效能好；之後再視情況升級。
- 世足賽氛圍：草皮綠、看台、球門、計分板字體。
- **足球造型必須是經典黑白五邊形 / 六邊形拼接（Telstar 樣式）**，非純色圓。
- 主畫面為三個模式的選擇入口（卡片式）。

### 2.2 共通功能
- 模式選單（首頁）
- 音效系統（見 2.5）＋ 全域靜音開關
- 分數顯示 HUD（見 2.4）與「最高分」本地儲存（localStorage）
- 結算畫面：本局分數、最高分、再玩一次、分享
- **分享圖卡**：把本局分數 / 模式產生成可分享圖片（Canvas 繪製 → 下載 / Web Share API）
- **多語系切換**：中／英（i18n 文案表，預設依瀏覽器語言）

### 2.3 背景（AI 生成圖）
- **背景採用 AI 生成的圖片素材（GPT / Gemini 生成）**，非純 Canvas 繪製，以避免失真、提升卡通寫實質感。
- 流程：用 AI 產出各場景背景（球場全景、球門正面、首頁等）→ 匯出為圖片放入 `/assets/bg/`，遊戲以 `<img>` / Canvas `drawImage` 作為底層繪製。
- 規格建議：
  - 直式主視覺 **1080×1920**、橫式 **1920×1080**，各備一版；以 `object-fit: cover` 自適應，重要元素（球門）留安全區避免被裁切。
  - 格式 **WebP**（壓縮率高、品質好），單張目標 < 300KB；提供 1x／2x 因應高解析螢幕。
  - 風格需一致（同一套卡通色調 / 光線），建議在同一輪 prompt 產出整組以維持一致性。
- 分層：背景圖（不動）＋ Canvas 前景（球、特效）＋ HTML HUD（分數），三層疊合。
- 各模式背景：
  - 模式一 顛球：球場 + 看台全景
  - 模式三 PK：球門正面視角（含球網）
  - 模式二 頭鎚：**背景為攝影機即時影像**，僅疊球門框線（不使用 AI 背景）
- 載入策略：背景圖預載並顯示 loading；失敗時退回單色／漸層底色保底。

### 2.4 計分 / HUD 顯示
- HUD 一律用 **HTML/CSS 疊在 Canvas 上層**（不畫進 Canvas）→ 文字清晰、好套多語系與 RWD。
- 各模式 HUD：
  - 顛球：`連續 N 下` + `最高分`（大數字置中上方，破紀錄時放大跳動 + 變色）
  - 頭鎚：`得分` + `剩餘時間進度條`
  - PK：**計分板**（雙方進球 ●○○○○ / 目前第幾球）
- 加分回饋：數字 **scale pop 彈跳動畫** + 對應音效。

### 2.5 音效
- **以 Web Audio API 合成基本音效**（彈球、加分、破紀錄等）；另備少量短音檔（踢球、進球網、群眾歡呼）放 `/assets/sounds/`。
- 統一 `SoundManager`：預載、音量控制、**全域靜音開關（狀態存 localStorage）**。
- ⚠️ 行動瀏覽器限制：需在使用者**首次手勢（點「開始」）後** unlock AudioContext 才能播放。
- 事件對應：命中／彈球、加分、破紀錄、進球、撲救、失誤、倒數。

### 2.6 技術選型建議
| 層面 | 建議 |
|------|------|
| 渲染 | 背景 AI 圖（drawImage）+ Canvas 2D 前景；模式 2 疊加 video + canvas |
| 物理 | 自製輕量物理（重力、速度、彈性、變形），不必引入完整物理引擎 |
| 攝影機追蹤 | **MediaPipe Tasks (Face / Pose Landmarker)** 或 TensorFlow.js，純前端、免後端 |
| 框架 | 可用原生 JS + Vite，或 React；遊戲迴圈用 `requestAnimationFrame` |
| 部署 | 靜態網站（Vercel / Netlify / GitHub Pages） |

## 3. 模式一：顛球挑戰（Keepy-Uppy）

### 3.1 玩法描述
球從上方落下，玩家用滑鼠點擊 / 手指點觸球，把球往上彈回去，盡量連續顛最多下不落地。

### 3.2 核心機制
1. **重力與落下**：球受重力加速度持續下墜。
2. **點擊回彈**：點到球時給球一個向上的速度。
3. **方向控制（重點）**：
   - 點擊位置相對球心的偏移量決定水平方向。
   - 點球體**右側** → 球往**左上**飛；點**左側** → 往**右上**飛（如同真的踢球側邊）。
   - 點越靠邊緣 → 水平速度越大、角度越斜。
   - 點越靠正中下方 → 越垂直、越高。
4. **變形動畫**：點擊瞬間球體沿撞擊方向**擠壓變形（squash）**，再彈回（stretch → 回正），模擬受力。
5. **計分**：每成功點一下 +1，球落到底線（落地）即結束。
6. **難度曲線**：隨分數提高，重力略增 / 球變小 / 加入微風水平擾動。

### 3.3 物理參數（初版可調）
- 重力 `g`：約 0.4~0.6 px/frame²
- 回彈基礎速度：依點擊力道固定值 + 隨機微擾
- 水平速度：`(球心x - 點擊x) * 係數`
- 邊界：左右牆反彈（可選），底線 = 結束

### 3.4 互動與手感細節
- 點到才有效，點空不計、不影響球。
- 命中時：變形 + 音效 + 短暫粒子 / 閃光回饋。
- 球體須有自轉（旋轉貼圖）增加真實感。

### 3.5 完成標準（驗收）
- [ ] 球為經典足球造型且會自轉
- [ ] 點擊位置確實影響飛行方向（左右、角度）
- [ ] 點擊時有明顯擠壓變形動畫
- [ ] 重力與回彈手感順暢（60fps）
- [ ] 連續計分、落地結束、最高分記錄

## 4. 模式二：頭鎚射門（攝影機 AR）

> 你提到不確定怎麼實現，以下給出推薦做法。

### 4.1 玩法描述
開啟前鏡頭，玩家出現在畫面中，虛擬足球從上方 / 前方飛來，玩家**用頭去頂球**，把球頂進球門得分。

### 4.2 技術實現建議（推薦方案）
**方案 A（推薦）：MediaPipe Face Landmarker / Pose Landmarker 偵測頭部位置**
- 用前鏡頭 `getUserMedia` 取得即時影像，畫在底層 `<video>`。
- 用 MediaPipe 偵測**頭部關鍵點（額頭 / 鼻子 / 兩眼中心）**，得到頭在畫面中的座標與大小。
- 在頭部位置放一個**看不見的碰撞區（額頭 hitbox）**。
- 虛擬球用 canvas 疊在影像上方落下，與 hitbox 碰撞時 → 觸發頂球，依碰撞點給球速度往球門飛。
- **加分判定**：偵測頭部**向上 / 向前的瞬間移動速度**，動作越大力道越強，避免「站著不動球自己撞到」。

**為什麼選這個**：純前端、不用後端、iOS Safari 也支援 `getUserMedia`，MediaPipe 在手機上效能可接受。

**方案 B（保底 / 降級）**：若裝置不支援或拒絕授權，退回「重力感應 / 點擊」版本，仍可玩，不阻擋使用者。

### 4.3 權限與隱私（重要）
- 進入模式前明確說明「需要攝影機，畫面僅在本機處理，不上傳」。
- 提供「拒絕 / 稍後」與降級玩法。
- 偵測授權失敗的引導文案（iOS 需在 HTTPS 下才可用攝影機）。

### 4.4 流程
1. 說明頁 → 請求攝影機權限
2. 校正：請玩家把頭移到框內
3. 倒數 → 球開始飛來
4. 頂中 → 球往球門飛 → 進球 / 撲救判定 → 計分
5. 時間到 / 失誤 N 次 → 結算

### 4.5 完成標準（驗收）
- [ ] 正確請求並處理攝影機權限（含拒絕情境）
- [ ] 能即時追蹤頭部位置並顯示在畫面
- [ ] 頂球碰撞判定準確、有方向與力道
- [ ] 影像不離開裝置（隱私聲明）
- [ ] 不支援裝置有降級玩法
- [ ] 行動裝置效能可玩（建議 ≥ 24fps）

### 4.6 風險
- iOS Safari 對相機 / 模型載入限制較多，需實機測試。
- 模型檔較大（首次載入時間），需 loading 與提示。
- 光線不足時偵測不穩，需提示玩家。

## 5. 模式三：PK 大賽（罰球對戰）

### 5.1 玩法描述
模擬 PK 罰球，兩個回合制：
- **回合 A — 玩家當射手**：玩家選方向 / 力道射門，**電腦當門將**會撲救。
- **回合 B — 玩家當門將**：**電腦射門**，玩家選方向撲救。
- 多輪累計比分，分高者勝（可做 5 球制 + 驟死賽）。

### 5.2 核心機制
**射門（玩家為射手）**
- 操作：拖曳 / 點擊選擇射門點（球門九宮格）+ 力道條（可選）。
- 球依選擇飛向球門，門將依 AI 撲向某方向。
- 命中且門將沒撲到該區 → 進球。

**撲救（玩家為門將）**
- 電腦選一個方向射門（有預備動作 / 提示）。
- 玩家點 / 滑向左中右（或九宮格）撲救。
- 方向一致 → 撲到（Save）；否則進球。

**電腦 AI**
- 門將：有反應時間與撲救機率，難度可調（簡單偏隨機、困難會預判玩家習慣）。
- 射手：權重隨機選角，難度越高越會挑刁鑽角度。

### 5.3 規則
- 標準 5 球，平手進入驟死（一人一球）。
- 計分板顯示雙方進球數與當前第幾球。

### 5.4 視覺
- 球門正面視角、門將動畫（左撲 / 右撲 / 中間 / 站立）。
- 進球 → 球網晃動 + 歡呼；撲救 → 撲救動作 + 觀眾反應。

### 5.5 完成標準（驗收）
- [ ] 射手回合：可選方向力道、球飛行、門將撲救判定
- [ ] 門將回合：電腦射門、玩家撲救判定
- [ ] 回合切換與 5 球 + 驟死賽流程
- [ ] 計分板正確、勝負結算
- [ ] 至少 2 種 AI 難度

## 6. 非功能需求
- **效能**：模式 1、3 維持 60fps；模式 2 ≥ 24fps。
- **載入**：首頁 < 2 秒（模式 2 模型可延遲載入）。
- **無障礙**：色彩對比足夠、文字可讀、提供靜音。
- **RWD**：手機 / 平板 / 桌機自適應。
- **安全**：HTTPS（攝影機必要）、不蒐集影像資料。
