# 球體擬真化準則（全模式通用）

> 基準實作：模式一顛球 `src/screens/keepy.js`（使用者認證「看起來就很像真的球」）。
> 任何模式的球都照這套規則做，PK 模式已於 2026-06-11 對齊。

## 1. 旋轉：旋轉量 = 位移 ÷ 半徑

球的自轉必須跟移動耦合，不可用固定角速度。

```js
// keepy.js（2D 畫面）：水平位移驅動旋轉，像真的在滾
b.rot += (b.vx * dt) / b.r

// pk.js（3D 投影）：改用「螢幕空間」位移與半徑，效果等價
b.rot += (screenX - b.prevSX) / screenR
```

效果：直射幾乎不轉（乾淨觸球）、香蕉球隨弧線明顯旋轉、彈地滾動方向自然正確。
向上/向下的移動不驅動 2D 旋轉（那是繞水平軸的旋轉，畫面上表現不出來）。

## 2. 影子：高度決定大小、模糊、透明度

越高 → 影子越**大**、越**模糊**、越**淡**；快落地 → 小而銳利深色。立體高度感主要靠這個。
（注意方向別做反：影子不是越高越小越深。）

```js
const hN = clamp(高度 / 參考高度, 0, 1)   // 0 = 貼地, 1 = 高
ctx.filter = `blur(${1 + hN * 8}px)`      // keepy 用 2 + hN*16
ctx.globalAlpha = 0.34 * (1 - hN * 0.7)
半徑 = 球地面投影尺寸 * (0.95 + hN * 0.95)
// 橢圓扁率：俯視草地約 0.24~0.36
```

## 3. 擠壓變形：阻尼彈簧，不用線性衰減

受擊瞬間沿飛行方向壓扁、回彈微過衝、收斂：

```js
b.sqv += 脈衝            // 受擊時（keepy 5.5、pk 4.5）
acc = -K * b.sq - C * b.sqv   // K≈900、C≈18
b.sx = 1 - b.sq          // 沿撞擊軸壓扁
b.sy = 1 + b.sq * 0.5    // 垂直略鼓（近似體積守恆）
```

## 4. 尺寸：一律世界座標（公尺）× 投影係數

畫面元素（球、影子、紅圈、特效圈）的像素尺寸都用 `公尺 × cam.K / rev.Kx` 換算，
**不可綁螢幕寬 `W`**——桌機寬螢幕會整個爆大（紅圈、衝擊波、出腳閃光都踩過這個雷）。

## 5. 球面繪製

`src/ball.js` 的 `drawBall()`：照片圓形裁切（或向量 Telstar 拼接）＋固定光源高光（不隨自轉）＋邊緣暗角。所有模式共用，不要另畫。
